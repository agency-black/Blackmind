## ERROR 001 — `DOCK-FULLSCREEN-PARTICLE-TARGET-STALE` ✅ CORREGIDO

**Ubicación exacta:** `index.html` · Función `onResize()` (líneas 1217-1232)

### Causa Raíz Identificada
El sistema Verlet utiliza `particles.dX.ty` como posición destino cuando el dock está abierto. Estos valores se inicializaban en `initParticles(cachedW, cachedH)` llamado solo una vez en el evento `window.load`. Al entrar a pantalla completa, `ResizeObserver` detectaba el cambio y llamaba `onResize()` que actualizaba `cachedH` pero **no actualizaba `p.ty`**. 

**Consecuencia:** Al abrir el dock en una ventana redimensionada, los átomos Verlet `dC2`/`dC3` (los centrales que forman la curva del dock) apuntaban a `h_original - 26` en lugar de `h_nuevo - 26`, apareciendo posicionados en el centro de la pantalla nueva.

### Corrección Aplicada

```javascript
const onResize = () => {
    const rect = wrapper.getBoundingClientRect();
    cachedW = rect.width; cachedH = rect.height;
    // Actualizar targets de partículas Verlet para la nueva dimensión
    // CORRECCIÓN: Los targets .ty deben actualizarse cuando cambia la altura
    // para evitar que el dock aparezca en posición incorrecta en fullscreen
    const bottom = cachedH - 1;
    const dockTop = cachedH - 26;
    particles.dL.ty = bottom;
    particles.dR.ty = bottom;
    particles.dC1.ty = bottom;
    particles.dC2.ty = dockTop;
    particles.dC3.ty = dockTop;
    particles.dC4.ty = bottom;
    updateGeometry(); startLoop();
};
```

### Verificación Post-Corrección
- ✅ La función `onResize()` ahora actualiza todos los targets `.ty` de las partículas Verlet
- ✅ Se preserva completamente la física Verlet (K_CENTER, K_LATERAL, DAMPING, DT)
- ✅ Se preserva la función `updatePhysics()` sin modificaciones
- ✅ Las curvas de Bézier funcionan correctamente tras el resize

## ERROR 002 — `TOOLBAR-HOVER-FLICKER-SVG-TRANSFORM-CONFLICT` ✅ CORREGIDO

**Ubicación exacta:** `index.html` · CSS (líneas 303-333) y SVG (líneas 823-856)

### Causa Raíz Identificada
El selector CSS original `#menu-bar-toolbar svg g:hover` capturaba **todos** los elementos `<g>` anidados dentro del SVG del toolbar. Los grupos `toolbar-share`, `toolbar-refresh` y `toolbar-action` tenían un atributo SVG de presentación `transform="translate(74, 0)"`. 

**Conflicto:** En el motor de renderizado de Chromium, el `transform` de presentación SVG y el `transform` CSS son propiedades de layer distintas. Al aplicar `transform: scale3d()` con `transform-box: fill-box` sobre un elemento que ya tenía un `transform` de presentación SVG, el bounding box calculado incluía el offset del `translate(74,0)`, desplazando el punto de transformación del `transform-origin: center` fuera del icono visible.

**Consecuencia:** Parpadeo errático y saltos visuales en el hover de los iconos del toolbar.

### Corrección Aplicada

**PASO 1 - CSS Selectores específicos por ID:**
```css
/* Toolbar icons hover - solo los grupos de primer nivel por ID */
#toolbar-search,
#toolbar-cube,
#toolbar-pdf,
#toolbar-share,
#toolbar-refresh,
#toolbar-action {
    cursor: pointer;
    transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1);
    transform-box: fill-box;
    transform-origin: center;
    will-change: transform;
}

/* Para los grupos con offset horizontal (share/refresh/action) */
#toolbar-share:hover {
    transform: translate(74px, 0) scale(1.167);
}
#toolbar-refresh:hover {
    transform: translate(74px, 0) scale(1.167);
}
#toolbar-action:hover {
    transform: translate(74px, 0) scale(1.167);
}

/* Para los grupos sin offset SVG */
#toolbar-search:hover,
#toolbar-cube:hover,
#toolbar-pdf:hover {
    transform: scale(1.167);
}
```

**PASO 2 - Eliminación de atributos SVG `transform="translate(74, 0)"`:**
```html
<!-- ANTES (con transform SVG de presentación): -->
<g id="toolbar-share" transform="translate(74, 0)">

<!-- DESPUÉS (sin transform SVG, offset manejado por CSS): -->
<g id="toolbar-share">
```

### Verificación Post-Corrección
- ✅ Los selectores CSS ahora son específicos por ID, evitando cascada en elementos anidados
- ✅ Los grupos con offset horizontal (share/refresh/action) usan `translate(74px, 0) scale(1.167)` en CSS
- ✅ Los grupos sin offset (search/cube/pdf) usan solo `scale(1.167)`
- ✅ Se eliminaron los atributos `transform="translate(74, 0)"` de los elementos SVG
- ✅ La funcionalidad de click en los iconos se preserva intacta
